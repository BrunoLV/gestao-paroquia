# Product Guidelines

## Prose Style
- **Tone**: Professional, clear, and direct.
- **Language**: Use clear and concise English for technical documentation and Portuguese (if applicable) for end-user facing messages.
- **Consistency**: Maintain a consistent terminology for calendar entities (e.g., Event, Project, Approval).

## UX Principles
- **Clarity**: API responses should be clear and informative, using standardized error codes.
- **Consistency**: Follow RESTful API conventions consistently across all endpoints.
- **Efficiency**: Optimize endpoints for common operations like calendar retrieval and filtering.
- **Security**: Always prioritize user privacy and secure data handling.

## Branding
- **Identity**: The API should reflect the official identity of the NS Fatima parish.
- **Terminology**: Use terms that are familiar to parish staff and members.
